COMPONENT 2: Gemma 4 e2b (intent + context classification model)
────────────────────────────────────────────────────────────────────────────
Name:        Gemma 4 e2b
Version:     4 (e2b variant, efficient 2B-class, ~7.2 GB quantized)
Source:      https://ollama.com/library/gemma4:e2b
License:     Gemma Terms of Use
             https://ai.google.dev/gemma/terms
Purpose:     Classifies prompt intent + context (research, creative writing,
             meta-discussion, work data sharing, code, personal, educational,
             ambiguous) and authoritatively sets the zone (green / amber /
             red) that drives pseudonymization and blocking.

             IronGate selected this model after a 166-scenario ground-truth
             benchmark against every plausible local alternative:
             - 162/166 accuracy (97.6%) on the IronGate context battery
             - 0 false positives on benign research/meta/creative/code
             - 1 false negative (news-style M&A paraphrase edge case)
             - 7.2 GB on disk (quantized)
             - 1.3s P50 / 1.6s P95 latency on M1 Pro / 16GB RAM
             - JSON schema compliance: 100% with Ollama format=json mode

Accepted fallback (unsupported, may be set explicitly via managed policy):
  - llama3.2:3b  (Meta, 2.0GB) — smaller footprint; accuracy against the
                  current classifier prompt is NOT validated
  - chrome-builtin (Chrome 138+ Gemini Nano, zero install) — available only
                  where Chrome's Prompt API is enabled

Models explicitly REJECTED for production:
  - gemma3:4b   (slow + inaccurate on this prompt shape)
  - phi3:mini   (no AMBER discrimination)
  - qwen2.5:3b  (false-alarm machine on the context battery)
  - all 1B models (insufficient nuance)

────────────────────────────────────────────────────────────────────────────
NETWORK BEHAVIOR
────────────────────────────────────────────────────────────────────────────
After installation in local-only mode, the installed components make these
network connections:

  Ollama service              → 127.0.0.1:11434 (localhost only, never external)
  IronGate extension          → 127.0.0.1:11434 (Ollama)
  IronGate extension (audit)  → Customer-controlled audit sink URL (S3/syslog/webhook)
                                 ONLY if auditLogDestination is set to a non-'none' value
  IronGate extension (policy) → Customer-controlled policyBundleUrl
                                 ONLY if policyBundleUrl is set
  IronGate extension          → AI tool platforms (chatgpt.com, claude.ai, etc.)
                                 — pseudonymized requests pass through here

In local-only mode, NO network call is made to:
  - irongate-api.onrender.com (IronGate's API server)
  - api.openai.com or any cloud LLM provider
  - Any IronGate-controlled domain

This is enforced architecturally by the Tier 2 adapter and verified by the
architecture invariant tests in the IronGate source repository.
