COMPONENT 2: Llama 3.2 3B (sensitivity classification model)
────────────────────────────────────────────────────────────────────────────
Name:        Llama 3.2 3B Instruct
Version:     3.2 (3B variant)
Source:      https://ollama.com/library/llama3.2:3b
License:     Llama 3.2 Community License Agreement
             https://github.com/meta-llama/llama-models/blob/main/models/llama3_2/LICENSE
Purpose:     Classifies AMBER-zone prompts (business confidentiality without
             direct PII) that the regex stack cannot handle.

             IronGate selected this model after benchmarking 9 candidates:
             - 28/30 accuracy on the IronGate scenario suite (93.3%)
             - 5/6 accuracy on AMBER_ZONE (vs 1/6 for regex alone)
             - 2.0 GB on disk
             - 1309ms P50 latency on M1 Pro / 16GB RAM
             - JSON schema compliance: 100% with Ollama format=json mode

Validated alternatives (also acceptable):
  - gemma4:e2b   (Google, 7.2GB, 93.3% accuracy)
  - chrome-builtin (Chrome 138+ Gemini Nano, zero install)

Models explicitly REJECTED for production:
  - gemma3:4b   (over-flags business confidentiality, 20/30 — worse than regex alone)
  - phi3:mini   (ties regex baseline with no AMBER improvement, 22/30)
  - qwen2.5:3b  (false-alarm machine, 12/30)
  - all 1B models (insufficient nuance for classification)

────────────────────────────────────────────────────────────────────────────
NETWORK BEHAVIOR
────────────────────────────────────────────────────────────────────────────
After installation in local-only mode, the installed components make these
network connections:

  Ollama service              → 127.0.0.1:11434 (localhost only, never external)
  IronGate extension          → 127.0.0.1:11434 (Ollama)
  IronGate extension (audit)  → Customer-controlled audit sink URL (S3/syslog/webhook)
                                 ONLY if auditLogDestination is set to a non-'none' value
  IronGate extension (policy) → Customer-controlled policyBundleUrl
                                 ONLY if policyBundleUrl is set
  IronGate extension          → AI tool platforms (chatgpt.com, claude.ai, etc.)
                                 — pseudonymized requests pass through here

In local-only mode, NO network call is made to:
  - irongate-api.onrender.com (IronGate's API server)
  - api.openai.com or any cloud LLM provider
  - Any IronGate-controlled domain

This is enforced architecturally by the Tier 2 adapter and verified by the
architecture invariant tests in the IronGate source repository.
